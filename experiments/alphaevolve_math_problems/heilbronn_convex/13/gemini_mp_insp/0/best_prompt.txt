MATHEMATICAL CONTEXT & THEORETICAL BACKGROUND:
- **PROBLEM COMPLEXITY**: The Heilbronn problem is among the most challenging in discrete geometry, with optimal configurations rigorously known only for n ≤ 4 points
- **ASYMPTOTIC BEHAVIOR**: For large n, the optimal value approaches O(1/n²) with logarithmic corrections, but the exact constant remains unknown
- **GEOMETRIC CONSTRAINTS**: Points must balance competing objectives:
  * Interior points can form larger triangles but create crowding
  * Boundary points avoid area penalties but limit triangle formation
  * Edge cases arise when three points become nearly collinear
- **SYMMETRY CONSIDERATIONS**: Optimal configurations often exhibit rotational symmetries (particularly 3-fold due to triangular geometry)
- **SCALING INVARIANCE**: The problem is scale-invariant; solutions can be normalized to any convex region
- **CRITICAL GEOMETRIC PROPERTIES**:
  * Delaunay triangulation properties and angle optimization
  * Voronoi diagram regularity as indicator of point distribution quality
  * Relationship between circumradius and triangle area
  * Connection to sphere packing and energy minimization principles

**Recommended implementation patterns**:
To achieve optimal point placement, you *must* implement a robust global optimization algorithm. Follow these steps:

1.  **Define the Search Space**: The problem is scale-invariant. For practical implementation, assume the convex region is the unit square `[0, 1] x [0, 1]`. All 13 points must have coordinates within this range. The optimization algorithm will search for the optimal `(x, y)` coordinates for all 13 points. This means the optimization variable will be a flattened array of 26 floating-point numbers.

2.  **Objective Function (`min_triangle_area_objective`)**:
    *   This function will accept a 1D numpy array `x` of 26 values (representing the 13 `(x, y)` coordinates).
    *   Reshape `x` into a `(13, 2)` numpy array of points.
    *   Iterate through all `C(13, 3) = 286` unique combinations of three points. `itertools.combinations` is useful here.
    *   For each triplet of points `(p1, p2, p3)`, calculate the area of the triangle they form. A numerically stable and efficient method is using the determinant formula: `area = 0.5 * abs(p1[0]*(p2[1]-p3[1]) + p2[0]*(p3[1]-p1[1]) + p3[0]*(p1[1]-p2[1]))`. This avoids `sqrt` until the very end and handles collinear points (area will be 0).
    *   Keep track of the minimum non-zero area found. If all points are collinear, the minimum area is 0.
    *   Since optimization algorithms typically *minimize*, the objective function should return the *negative* of the smallest triangle area found. This effectively maximizes the minimum area.
    *   **Acceleration**: For performance, consider using `numba.jit` to compile this objective function or critical sub-parts of it, especially the triangle area calculation loop.

3.  **Optimization Engine**:
    *   **PRIMARY RECOMMENDATION**: `scipy.optimize.differential_evolution` is highly recommended for this problem due to its global search capabilities, robustness to local minima, and suitability for non-smooth objective functions (like `min()`).
    *   Set the `bounds` for each coordinate to `(0.0, 1.0)`. This will be 26 bounds for the 26 variables.
    *   Ensure reproducibility by setting the `seed` parameter (e.g., `seed=42`).
    *   Consider tuning `popsize` and `maxiter` within the `eval_time` constraints. A larger `popsize` can explore the search space more thoroughly but takes longer.
    *   Alternative: `scipy.optimize.basinhopping` can also be effective, potentially combined with a local minimizer like `L-BFGS-B`.

4.  **Initial Point Generation**:
    *   **Low-discrepancy sequences**: Utilize `sobol_seq` (from `sobol_seq` package) or other quasi-random sequences to generate a good initial population for `differential_evolution`. This provides more uniform coverage of the search space than purely random points.
    *   Consider using configurations with rotational symmetry (e.g., 3-fold) as initial guesses or components of the initial population, as optimal solutions often exhibit such symmetries.

5.  **Return Value**: The constructor function should return the `x` (flattened coordinates) corresponding to the best minimum area found by the optimizer, reshaped back into a `(13, 2)` numpy array.
